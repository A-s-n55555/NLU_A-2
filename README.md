# Part 1: Learning Word Embeddings from IIT Jodhpur Data

This project processes a corpus of PDF documents, performs extensive Natural Language Processing (NLP) text cleaning, and trains custom **Continuous Bag of Words (CBOW)** and **Skip-Gram (with Negative Sampling)** Word2Vec models from scratch using NumPy. It concludes with semantic analysis, analogy testing, and visualizations using PCA and t-SNE.

## 🚀 Features
* **PDF Data Extraction:** Automatically reads and extracts text from multiple PDF files using `pdfplumber`.
* **Advanced Text Preprocessing:** Removes boilerplate text, page numbers, metadata (URLs, ISBNs), stop words, and punctuation.
* **Custom Word2Vec Implementation:** * CBOW model trained using forward/backward propagation via NumPy.
  * Skip-Gram model trained using Negative Sampling.
* **Semantic Evaluation:** Supports calculating cosine similarities and solving word analogies (e.g., *a is to b as c is to ?*).
* **Visualizations:** Generates a WordCloud of the corpus and 2D plots (PCA & t-SNE) of the learned word embeddings.

# Part 2: Character-Level Name Generation Using RNN Varients
This project is a from-scratch implementation of Character-Level Language Models using only NumPy. It trains on a dataset of names to learn character sequences and generates completely new, original names.

## 🚀 Features
* **Three Custom Architectures:**
  * Vanilla Recurrent Neural Network (RNN)
  * Bidirectional Long Short-Term Memory (BiLSTM)
  * RNN with Basic Dot-Product Attention
* **From-Scratch Math:** No PyTorch or TensorFlow; all forward passes, Backpropagation Through Time (BPTT), and Adagrad/SGD gradient updates are written in pure NumPy.
* **Performance Metrics:** Automatically calculates **Novelty Rate** (percentage of generated names not in the training set) and **Diversity Ratio** (uniqueness of generated names).
* **Visualizations:** Generates a matplotlib graph comparing the training loss of all three models over the epochs.

## 🛠️ Prerequisites & Installation

This script requires Python 3.7+ and two external libraries. 

Open your terminal or command prompt and run:
`pip install numpy matplotlib`

## 📁 Dataset Setup

To train the models, you must provide a training dataset:
1. Create a text file named `TrainingNames.txt` in the exact same directory as the script.
2. Populate this file with a list of names, placing one name per line. 

*Note: The script will automatically convert all text to lowercase and build a character-level vocabulary from this file.*

## ▶️ How to Run the Project

Once your `TrainingNames.txt` file is in place, execute the script from your terminal:

`python nlu_assignment_2_2_.py`

### 📊 Expected Output Workflow

When you run the script, it will execute in the following sequence:
1. **Training:** It will train the Vanilla RNN, BiLSTM, and RNN+Attention models sequentially, printing the loss for each epoch.
2. **Loss Plot:** A matplotlib window will pop up showing the loss curves for all three models. *(Note: You must close this plot window for the script to proceed to the generation phase).*
3. **Sample Generation:** The console will print out a few sample names generated by each model, starting with specific seed characters (e.g., 'a', 'd', 'i', 'e', 'm', 'r').
4. **Metrics:** A formatted table will be printed to the console displaying the Novelty Rate and Diversity Ratio for each architecture.
5. **Export:** Finally, the script will generate 100 new names using each model (300 total) and save them to a local file named `generated_names.txt`.

## ⚙️ Hyperparameter Tuning
If you want to experiment with the learning capacity of the models, you can adjust the following variables directly in the script:
* `hidden_size`: The dimensions of the hidden state (Default: 120 for RNN, 128 for LSTM/Attention).
* `lr` / `learning_rate`: The step size for gradient descent (Default: 0.02 for RNN/LSTM, 0.002 for Attention).
* `epochs`: Number of training iterations (passed into the `train_model` function).
* `length`: The maximum character length of the generated names.
